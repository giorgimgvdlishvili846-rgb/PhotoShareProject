# 📸 PhotoShare - ფოტოების გაზიარების ვებ-პლატფორმა (CMS)

PhotoShare არის თანამედროვე, უსაფრთხო და ობიექტზე ორიენტირებული (OOP) ვებ-აპლიკაცია, რომელიც შექმნილია ფოტოების ასატვირთად, სამართავად და გასაზიარებლად. პროექტი სრულად აკმაყოფილებს ვებ-პროგრამირების ფინალური პროექტის ყველა კრიტერიუმს (Front-end და Back-end ნაწილებს).

---

## 🚀 ფუნქციონალური მოთხოვნების შესრულება (სილაბუსის მიხედვით)

### 🎨 Front-end ნაწილი (15 ქულა)
* **სტრუქტურა და სემანტიკა:** საიტი შედგება 7 სემანტიკურად სწორად აგებული გვერდისგან (`index`, `register`, `login`, `profile`, `gallery`, `admin`, `logout`). გამოყენებულია HTML5-ის სემანტიკური ტეგები (`<header>`, `<main>`, `<footer>`, `<section>`).
* **ცენტრალიზებული CSS3:** მთელი პროექტის სტილიზაცია იმართება ერთიანი `assets/css/style.css` ფაილიდან.
* **Responsive დიზაინი:** საიტი სრულად ადაპტირებადია (რესფონსივი) ნებისმიერი ზომის ეკრანისთვის CSS Grid-ის (`repeat(auto-fill, minmax(280px, 1fr))`) და მედია რექვესთების (`@media`) გამოყენებით.
* **ვიზუალური ეფექტები (CSS3):**
  * `Background` & `Gradient` (თანამედროვე გრადიენტები ფორმებსა და ღილაკებზე).
  * `Box-shadow` (ჰაეროვანი ჩრდილები ინტერფეისის სიღრმისთვის).
  * `Transitions` & `Transform` (ანიმაციური რეაგირება ელემენტებზე მაუსის მიტანისას - Hover ეფექტები).
* **ფერთა თეორია:** ფერები შერჩეულია `Color Wheel`-ზე დაყრდნობით (პრემიუმ მუქი ლურჯი და აქცენტირებული ცოცხალი ლურჯი) HEX და RGBA ფორმატების გამოყენებით.
* **JQuery ვალიდაცია:** რეგისტრაციისა და ავტორიზაციის ფორმებზე რეალიზებულია კლიენტის მხარეს ვალიდაცია `validation.js`-ში JQuery-ის საშუალებით, რაც აუმჯობესებს UI/UX-ს.

### ⚙️ Back-end ნაწილი (15 ქულა)
* **ობიექტზე ორიენტირებული პროგრამირება (OOP):** ბექენდი მთლიანად აგებულია PHP კლასებზე (`Database`, `User`, `Post`, `Logger`).
* **უსაფრთხო ავტორიზაცია/რეგისტრაცია:** მომხმარებლის რეგისტრაცია მიბმულია უნიკალურ ელ-ფოსტაზე. პაროლები დაცულია და დაჰეშირებულია `password_hash()`-ის (BCRYPT) ალგორითმით.
* **უსაფრთხო ფაილის ატვირთვა:** ფოტოების ატვირთვისას სისტემა ამოწმებს ფაილის გაფართოებას (JPG, PNG, GIF) და ანიჭებს უნიკალურ სახელს (`uniqid()`) ფაილური სისტემის უსაფრთხოებისთვის.
* **როლებზე დაფუძნებული წვდომა (RBAC):** სისტემაში არსებობს მომხმარებლის 2 როლი: `Admin` და `User`. უფლებები კონტროლდება სესიებით (`$_SESSION`).
* **ჩანაწერების CRUD მართვა:** ჩვეულებრივ მომხმარებელს შეუძლია მხოლოდ საკუთარი ფოტოების წაშლა, ხოლო `Admin`-ს აქვს სრული კონტროლი ნებისმიერი მომხმარებლის პოსტის წაშლაზე.
* **ფაილური სისტემის ლოგირება:** შექმნილია ლოგერი, რომელიც ყოველ აქტივობას (რეგისტრაცია, ავტორიზაცია, ფოტოს ატვირთვა/წაშლა) აღრიცხავს სპეციალურ ფაილში. ადმინს აქვს ამ ლოგების ნახვის უფლება თავის პანელში. გარედან ფაილზე წვდომა ბლოკირებულია `.htaccess` ფაილით.

## 📊 მონაცემთა ბაზის არქიტექტურა (4 ცხრილი)

მონაცემთა ბაზა შეიცავს 4 ცხრილს, რაც უზრუნველყოფს სილაბუსით მოთხოვნილ კავშირების ტიპებს:
1. `users` - ინახავს მომხმარებლის ძირითად მონაცემებს.
2. `roles` - ინახავს სისტემის როლებს (Admin, User).
3. `user_roles` - შუამავალი ცხრილი, რომელიც უზრუნველყოფს **N:N (Many-to-Many)** კავშირს მომხმარებლებსა და როლებს შორის.
4. `posts` - ინახავს ატვირთულ ფოტოებს. `user_id` სვეტის საშუალებით მყარდება **1:N (One-to-Many)** კავშირი `users` ცხრილთან (ერთ მომხმარებელს აქვს ბევრი ფოტო).
